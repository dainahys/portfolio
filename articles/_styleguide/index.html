Design System

Article Styleguide

모든 아티클이 공유하는 컴포넌트 라이브러리. 새 글은 /articles/_template/를 복사하고, 본문 블록은 여기서 스니펫을 가져다 쓴다. 톤을 바꾸려면 /articles/assets/article.css 한 파일만 고친다.

v1.0 · dainahys.com

Workflow

새 아티클 추가하는 법

  1. 폴더 복사. /articles/_template/를 /articles/<topic>/<slug>/index.html로 둔다. (예: articles/braze/survey-modal/)
  2. 플레이스홀더 채우기. {{TITLE}} {{DESC}} {{SLUG_URL}} {{DATE}} 등과 JSON-LD의 datePublished·FAQ 문항을 글 내용에 맞게 수정.
  3. 본문 작성. 아래 컴포넌트 스니펫을 복사해 블록을 채운다. 링크·캐노니컬은 항상 루트절대경로 / 기준.
  4. 목록 카드 추가. 홈 index.html의 Articles 섹션(KO·EN)에 카드 1장 추가.

Themes

브랜드 테마 — 글마다 액센트 1개

컴포넌트는 모두 --accent* 토큰만 참조한다. 글 최상위(<body>)에 data-theme 한 줄로 액센트가 일괄 전환된다. 기본(미지정)은 그 외/promotion = 중립 잉크.

테마data-theme--accent쓰임
그 외 / promotion(없음)#1F1F2B 중립일반·프로모션 글(기본값)
숨고soomgo#6A3AF2 보라숨고 관련 글
넥스트유니콘nextunicorn#0064FF 블루넥스트유니콘 관련 글
▌ 적용 · 글 최상위에 한 줄
<body data-theme="soomgo">        // 숨고 글 → 보라
<body data-theme="nextunicorn">  // 넥유 글 → 블루
<body>                           // 그 외 → 중립(기본)

인라인 SVG .diagram은 fill/stroke에 currentColor 또는 var(--accent)를 쓰면 테마를 따라온다(하드코딩 hex 지양).

Tokens

색 · 타이포

--accent --accent-2 --accent-tint --bg --ink --code-bg

액센트는 테마 토큰(--accent*). 그 외 base = --bg #fcfbf9 · --ink #1a1a1a · --muted · --line (테마 무관). Display = Poppins · Body = Pretendard · Mono = DM Mono · 본문폭 720px

prose · 본문

기본 문단. 강조는 <strong>(굵게), <em>(보라 키워드), .lead(굵은 리드), 인라인 <code>.

리드 문장. 일반 본문이고 핵심은 굵게, 중요 키워드는 보라로, 코드값은 external_user_id처럼 표기한다.

ul.hyp · 라벨 불릿

가설·항목처럼 "라벨 + 설명" 나열에. 좌측 보라 보더 + mono 라벨.

  • H1 노출도달이 낮아 인지하지 못했다
  • H2 가치혜택 매력이 부족했다

ol.reasons · 번호 리스트

이유·단계처럼 순서 있는 나열에. 보라 번호 배지 + 굵은 리드.

  1. 대상을 행동으로 정의했다. 활동 빈도·참여 패턴으로 세그먼트를 도출했다.
  2. 편향을 설계로 막았다. 1차는 리워드 없이 돌렸다.

.mtable · 비교/매핑 표

세그먼트×가설처럼 본문용 매트릭스. sg-tint 헤더, 첫 열은 mono 보라.

세그먼트정의가설
A 미인지본 적 없음노출 (H1)
B 참여자참여함가치 (H2)

.dtable · 데이터 추출 표

쿼리 결과·로그처럼 raw 데이터. mono 폰트, 가로 스크롤. 긴 JSON 셀은 class="j".

EXTERNAL_IDEVENT_TIMEPROPERTIES
u_10··abcd2026-02-23{"segment":"A", …}

.code · 코드블록

라벨(.code-label) + <pre>. 주석은 .cm. <·>는 &lt;·&gt;로 이스케이프.

▌ Braze 쿼리 빌더 · SQL
SELECT external_user_id AS external_id
FROM   USERS_BEHAVIORS_CUSTOMEVENT_SHARED
WHERE  name = 'promo_survey_completed'  // 완료 이벤트

figure.diagram · SVG 다이어그램

플로우·구조도. 인라인 <svg> + <figcaption>. 보더·라운드 자동.

.callout / .takeaway / .pull · 강조 박스

callout=보조 노트, takeaway=핵심 정리, pull=문장 punchline.
